🎯 Agent 协同团队

不是单打独斗 — 是一个有分工、有契约、有监督的虚拟研发团队

🎯

Orchestrator 主调度器

需求探索 + 模式选择 + 全局调度。根据任务复杂度自动选择轻量/顺序/监督者模式，全程无需人工确认

📋

Planner 开发代表

技术方案设计 → 任务拆解 → 产出共享契约文件。以契约文件为前后端+测试的共同基准，消除理解偏差

🔧

Backend + Frontend Dev

无依赖任务并行调度执行。各自读取同一份契约文件，确保后端字段名 ↔ 前端字段名精准对齐

🔍

Code Auditor 代码审计师

编译检查 + 构建验证 + 引用完整性审计。发现错误直接通知对应开发 agent 修复，不等待全流程结束

🧪

API Tester 接口测试

契约验证 + 功能测试 + 异常测试。逐字段对比实际响应与契约定义，简单 Bug 直接修复并重跑

📦

Archiver 归档验收

全流程通过后自动归档 Spec、Change、Tasks。完整可审计的开发轨迹，一次触发全程记录

/orchestration → Plan → Dev (并行) → Audit → Test → Archive

⚡ Dynamic Workflows

不是固定死板的流水线 — Orchestrator 根据任务复杂度自动选择最优调度策略

🚀 轻量模式

单模块小改动
  • 预估 < 5 分钟
  • 跳过 Plan + 审计 + 测试
  • 直接调度 Dev agent
  • 完成后简要汇报变更
  • 零额外开销、最快响应

📋 顺序流水线

需求明确
  • 需求清晰、≤ 2 个模块
  • 固定 6 阶段全流程
  • 前后端无依赖时并行
  • 阶段间自动推进无确认
  • 失败自动重试，最多 3 次

🧠 监督者模式

复杂 / 多模块
  • 跨领域 / 不确定性强
  • 动态任务图 + 实时调度
  • Agent 间可直接通信
  • 增量验证 + 动态重规划
  • 并行机会自动识别利用

Orchestrator 模式选择决策树

任务进入 → 预估复杂度分析
├── 单模块 + <5min → 轻量模式
├── 需求明确 + ≤2模块 → 顺序模式
└── 复杂/多模块/不确定 → 监督者模式

🔄 Fast Fix + Auto CI/CD

云端友好：发现 → 修复 → 验证 → 归档，全程自动闭环

🩹 Fast Fix 自愈机制

测试阶段发现字段不一致、接口偏差等简单问题时，测试 Agent 直接进入修复→重跑循环，不抛出阻塞。只有无法自动修复的复杂问题才记录到报告。

✨ 核心 设计优势

01

Spec-Driven 全链路

explore → propose → apply → archive。每个变更都有完整的设计文档、契约文件和任务追踪。需求→代码的映射可追溯。

02

Context Passing Chain

探索结果 → 设计方案 → 契约文件 → 开发报告 → 测试报告。每个 Agent 拿到的是结构化上下文，不是模糊的需求描述。

03

Contract-First 工程化

契约文件是 Backend / Frontend / Test 三个 Agent 的 Shared Truth。逐字段定义（字段名、类型、枚举值），消除各自理解的偏差。

04

Parallel Agent 并行调度

无依赖任务自动并行。监督者模式动态识别并行机会。前后端同时开发、多子任务并发执行，最大化吞吐量。

05

Zero-Confirmation Flow

全流程自动推进不询问。只在需求存在歧义、Agent 返回 NEEDS_CLARIFICATION、或重试 3 次失败时才与用户交互。其余时间静默高效。

06

Unified Process Logging

所有 Agent 执行过程追加写入同一日志文件。时间戳 + Agent 名称 + 事件描述。完整的、可审计的、可回溯的执行轨迹。

07

Graceful Degradation

失败不阻塞全流程。简单 Bug 自修复，复杂问题记录继续。DONE_WITH_CONCERNS 不阻塞归档。只有重试耗尽才标记 FAILED。

08

Retry & Rollback 安全网

最大 3 次重试 + 指数退避。失败快照保存中间状态。紧急回滚 git checkout。Agent 之间可互相请求协助，不依赖主调度器单点决策。

🔗 Context Passing Chain

信息在各阶段间不丢失、不变形 — 每个 Agent 拿到的是精确的结构化上下文
