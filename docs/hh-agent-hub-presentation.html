端到端 SDLC 流程

从需求到部署，7 种 Agent 角色接力完成，全流程可追踪

📋 Stage 1 — 需求分析 Agent
解析需求文档，提取功能点、约束条件，输出结构化需求规格
📐 Stage 2 — 方案设计 Agent (Claude Code)
阅读现有代码，设计技术方案，写入 workspace 供下游使用
💻 Stage 3 — 编码开发 Agent (Claude Code)
读取设计方案，在同一份代码仓库上开发，生成变更文件清单
🔍 Stage 4 — 代码审查 Agent (Reasonix)
低成本代码分析，检查规范、安全、性能问题
🧪 Stage 5 — 质量验证 Agent (a2a)
调用自动化测试服务，跑测试用例，输出测试报告
🛑 Stage 5.5 — 人工确认中断
所有检查通过后，触发人工审批确认，防止自动部署风险
🚀 Stage 6 — CI/CD 部署 Agent
构建 → 部署 → 健康检查 → 通知结果

智能触发器 — 企微机器人

业内唯一支持企业微信智能机器人 WebSocket 长连接的 Agent 平台，群聊/私聊即触发

🔗 握手绑定 — 完整交互协议

🖥️ hh-agent-hub 管理平台
Step 1 创建触发器 → 选择应用

用户选择关联的 App，点击"生成绑定验证码"

Step 2 POST /wecom-bot/generate-code

后端生成 6 位随机码 A3K9X2，写入 Redis wecom_bind:A3K9X2，TTL 300s

Step 3 前端轮询等待 ⌛

每 5 秒 GET /bind-status/A3K9X2 → Redis 查询 wecom_bind_result:A3K9X2

Step 5 绑定成功 ✓

轮询查到 wecom_bind_result:A3K9X2 → 自动填充 chat_type + chat_id → 保存触发器

Step 6 Redis pub/sub 热更新

发布 wecom_bot:refresh → wecom-bot 容器无重启加载新映射

Step 4 用户在企微发送验证码

① aibot SDK 收到 WebSocket 消息回调
② 检查消息是否为纯验证码格式
③ 命中 wecom_bind:A3K9X2 → 提取 chat_type + chat_id
④ 写入 wecom_bind_result:A3K9X2，匹配成功 🎯
⑤ 回复企微："绑定成功！Chat ID: xxx"
