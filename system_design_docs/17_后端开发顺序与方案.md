# 后端开发顺序与方案

## 总体策略

**自底向上，逐层验证**：基础设施 → 数据层 → 业务层 → 接口层 → 集成层。

每个阶段完成后有明确的**验证标准**，通过后才能进入下一阶段。

## 阶段总览

```
Phase 0: 项目骨架         (0.5天)
Phase 1: 数据层           (2天)
Phase 2: 认证与用户       (1.5天)
Phase 3: 资源目录 CRUD    (2天)
Phase 4: Agent 核心       (3天)
Phase 5: 工作流引擎       (3天)
Phase 6: Chat API 与集成  (2天)
Phase 7: 可观测性         (1天)
───────────────────────────────
合计: 15 天
```

---

| 批次 | 模型 | 依赖 |

| 2 | Agent | User |
| 3 | McpServerRegistry, KnowledgeBase, SkillRegistry | User |
| 4 | AgentMcpLink, AgentKbLink, AgentSkillLink, ContentBlock | Agent + McpServer/KB/Skill |
| 5 | Workflow | User + Agent |
| 6 | App | User + Workflow |
| 7 | Session | App |
| 8 | WorkflowTrace | Workflow + App |
| 9 | AuditLog | User |

**验证**：
- `aerich init-db` 成功
- 启动时 Tortoise 连接 MySQL 无报错
- Admin 用户和默认配置自动插入

## Phase 2: 认证与用户 (1.5天)

**产出**：登录/注册/Token 刷新/API Key 验证

```
api/
├── auth.py              # POST /auth/login, POST /auth/register, POST /auth/refresh
├── deps.py              # get_current_user 依赖, require_admin 依赖
├── middleware.py         # ApiKeyMiddleware (Chat API 用)
utils/
├── crypto.py            # Fernet 加密 + bcrypt 密码哈希
└── jwt.py               # JWT 签发 + 验证
```

**API 清单**：

| 方法 | 路径 | 权限 |
|------|------|------|
| POST | `/api/v1/auth/login` | 公开 |
| POST | `/api/v1/auth/register` | 公开(开发期) / admin(生产) |
| POST | `/api/v1/auth/refresh` | Bearer Token |

**验证**：
- 登录成功返回 JWT
- 用 JWT 访问受保护端点成功
- 用 API Key 访问 Chat API 成功
- 过期 Token 被拒绝(401)
- 非 admin 访问系统设置被拒绝(403)

## Phase 5: 工作流引擎 (3天)

**产出**：三种编排模式 + 错误处理 + 并行 + workspace

```
core/
├── workflow_engine.py   # WorkflowEngine (supervisor/sequential/graph)
├── workflow_executor.py # WorkflowExecutor (timeout/retry/skip)
├── workspace_manager.py # WorkflowWorkspaceManager
└── parallel_executor.py # execute_parallel_group
api/
├── workflows.py         # 9 个端点
```

**开发顺序**：StateGraph 基础 → sequential → supervisor → graph → 错误处理 → 并行 → workspace

**验证**：
- Sequential: 3 个 Agent 顺序执行，上游输出传递给下游
- Supervisor: 主管正确路由到对应 Worker
- 错误: fail_fast 终止、skip_continue 跳过、retry 重试
- 并行: 两个 Agent 同时执行，取最长耗时
- Workspace: 目录正确创建，context.json 正确更新
- 人工中断: 工作流正确暂停，resolve 后继续

| Redis | 7 | 缓存/限流 |
| Node.js | 20+ | MCP Server (npx) |

## 本地开发启动

```bash
# 1. 创建数据库
mysql -u root -e "CREATE DATABASE agent_platform CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 2. 安装依赖
pip install -r requirements.txt

# 3. 初始化数据库
aerich init -t backend.config.TORTOISE_ORM --location backend/migrations
aerich init-db

# 4. 启动服务
uvicorn backend.main:app --reload --host 0.0.0.0 --port 8000
```
