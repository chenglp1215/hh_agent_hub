# 前端开发顺序与方案

## 总体策略

**从骨架到页面，逐步接入后端**：先搭框架能跑 → 再做核心页面 → 最后接后端 API。

## 阶段总览

```
Phase 0: 项目骨架         (0.5天)
Phase 1: 布局与路由       (1天)
Phase 2: 资源目录管理     (2天)
Phase 3: Agent 管理       (2天)
Phase 4: 工作流编辑器     (2天)
Phase 5: 应用管理与测试   (1.5天)
Phase 6: 监控与追踪       (1.5天)
Phase 7: 主控台与收尾     (1天)
───────────────────────────────
合计: 11.5 天
```

---

## Phase 0: 项目骨架 (0.5天)

```bash
npm create vite@latest frontend -- --template vue-ts
cd frontend
npm install ant-design-vue @ant-design/icons-vue
npm install vue-router pinia axios
npm install @vue-flow/core @vue-flow/background @vue-flow/controls
npm install monaco-editor
npm install tailwindcss @tailwindcss/vite
```

**产出**：Vite 项目可启动，Ant Design Vue 全局注册，Pinia 初始化，axios 封装，空路由占位。

**验证**：`npm run dev` 启动，浏览器打开看到空白首页。

| 页面 | 功能 |

**组件**：
| 组件 | 说明 |
|------|------|
| `McpServerSelector.vue` | 从资源目录勾选 MCP Server + 工具过滤 |
| `KnowledgeBaseSelector.vue` | 从资源目录勾选知识库 |
| `SkillsSelector.vue` | 从资源目录勾选 Skill |

**验证**：

## Phase 4: 工作流编辑器 (2天)

**产出**：Vue Flow 画布 + 节点拖拽 + 属性配置

| 页面 | 功能 |
|------|------|
| `WorkflowList.vue` | 搜索 + 类型筛选 + 版本号 + 状态(draft/published) |
| `WorkflowEditor.vue` | 左侧 Agent 面板 → 中央 Vue Flow 画布 → 右侧属性面板 |

**Vue Flow 节点类型**：
| 节点 | 外观 | 配置 |
|------|------|------|
| 主管 Agent | 蓝色圆角矩形 | 选择 Agent + Supervisor Prompt |
| 工作 Agent | 绿色圆角矩形 | 选择 Agent + 阶段名 |
| 条件判断 | 菱形 | 条件字段 + 条件值 |
| 并行组 | 虚线边框容器 | 组内 Agent 列表 + 合并策略 |
| 人工中断 | 红色标记节点 | 中断触发条件 + 默认动作 |
| 开始/结束 | 圆形 | 无 |

**验证**：
- 从左侧拖拽 Agent 到画布
- 连线连接节点
- 选中节点右侧面板显示属性
- 保存工作流成功
- 发布后状态变为 published
- validate 接口检测死循环/断连

## Phase 6: 监控与追踪 (1.5天)

**产出**：执行追踪列表 + 详情时间轴 + WebSocket 实时更新

| 页面 | 功能 |
|------|------|
| `ExecutionTraceList.vue` | 搜索 + 状态筛选 + 表格(execID/工作流/状态灯/耗时/时间) + 自动刷新 |
| `ExecutionTrace.vue` | 三个 Tab(概览/时间轴/原始数据) + WebSocket 实时模式 |

**时间轴组件**：
```
ExecutionTimeline.vue
  ├── AgentSpan.vue       # Agent 级别的条(可展开)
  │   ├── LlmSpan.vue     # LLM 调用(紫色)
  │   ├── ToolSpan.vue    # MCP 工具(蓝色)
  │   └── KbSpan.vue      # 知识库检索(青色)
  └── SpanDetail.vue      # 点击展开详情面板
```

**验证**：
- 工作流执行后追踪列表中出现记录
- 点击进入详情，时间轴正确渲染
- 运行中的工作流 WebSocket 实时更新
- 展开 Agent 可看到 LLM Prompt/Response + 工具参数/结果
- 导出 JSON 文件

## 技术要点

### 设计系统
遵循 `DESIGN.md` (Linear)：
- 暗色画布 `#010102`
- 强调色 `#5e6ad2`
- 字体: Linear Display / Text / Mono 回退 SF Pro
- 圆角: xs=4 / sm=6 / md=8 / lg=12

### API 调用分层
```
views → stores (Pinia) → api/*.ts (axios) → Backend
  │         │
  └─────────┘ (组件不直接调 API)
```

### 说明
- 全部使用 Mock 数据驱动开发(Mock Service Worker 或简单 JSON)
- Phase 2-6 可独立开发，不依赖后端
- 后端就绪后替换 Mock 为真实 API
