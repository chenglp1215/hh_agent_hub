# 多容器部署方案

## 一、容器拓扑

```
                         ┌──────────────────────┐
                         │      Client           │
                         └──────────┬───────────┘
                                    │ :80
                                    ▼
┌──────────────────────────────────────────────────────────────┐
│  frontend (Nginx)                                            │
│  - 静态文件服务 (Vue dist)                                    │
│  - /api/*     → main:8000                                    │
│  - /ws/*      → main:8000 (WebSocket)                        │
└───────────────────────────┬──────────────────────────────────┘
                            │
              ┌─────────────┼─────────────┐
              ▼             ▼             ▼
┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  main        │  │  worker-1    │  │  worker-N    │
│  FastAPI     │  │  工作流执行   │  │  ...         │
│  CRUD API    │  │  MCP Client  │  │              │
│  WebSocket   │  │  LangGraph   │  │              │
└──────┬───────┘  └──────┬───────┘  └──────┬───────┘
       │                 │                 │
       └─────────┬───────┴─────────┬───────┘
                 │                 │
                 ▼                 ▼
         ┌──────────┐    ┌──────────────┐
         │  MySQL   │    │    Redis     │
         │  :3306   │    │    :6379     │
         └──────────┘    └──────────────┘
                 │                 │
                 ▼                 ▼
         ┌──────────────────────────────────┐
         │  workspace_data (共享 Volume)     │
         │  /data/workflow_workspaces/      │
         └──────────────────────────────────┘
```

## 二、容器职责

| 容器 | 职责 | 扩缩 | 端口 |
|------|------|------|------|
| **frontend** | Nginx 静态文件 + 反向代理 | 1 实例 | 80 |
| **main** | FastAPI: CRUD API, Auth, WebSocket 推送 | 1-2 实例 | 8000 (内部) |
| **worker** | 工作流执行(LangGraph), MCP 进程管理, 追踪采集 | 1-N 实例 | 无 |
| **mysql** | 持久化存储(15 张表) | 1 实例 | 3306 |
| **redis** | 缓存, 会话, 限流, main↔worker 消息 | 1 实例 | 6379 |

### 为什么拆 main 和 worker？

| 问题 | 不拆 | 拆分后 |
|------|------|--------|
| 工作流执行耗时长(分钟级) | 阻塞 FastAPI 事件循环 | Worker 独立执行，API 不受影响 |
| 并发工作流多 | API 响应变慢 | Worker 水平扩展 |
| 长耗时 Agent 执行 | 阻塞 API 事件循环 | Worker 独立执行 |
| 部署/重启 | 改 API 需要中断正在跑的工作流 | 各自独立部署 |

## 三、main ↔ worker 通信

```
POST /chat ──▶ main
                │
                ├── 1. 验证 API Key
                ├── 2. INSERT workflow_traces (status=pending)
                ├── 3. REDIS PUBLISH execution:new {execution_id}
                └── 4. 返回 { execution_id, status: "accepted" }

              worker (polling + subscribe)
                │
                ├── 启动时 SUBSCRIBE execution:new
                ├── 收到消息 → 拉取 execution 详情
                ├── 执行工作流 (LangGraph)
                ├── 写 workspace artifacts + trace.json
                ├── UPDATE workflow_traces (status/result)
                └── REDIS PUBLISH execution:update {execution_id, span, ...}

              main (WebSocket handler)
                │
                ├── SUBSCRIBE execution:{id}:update
                ├── 收到 update → ws.send_json(event)
                └── 客户端实时看到执行进度
```

### Redis 频道设计

| 频道 | 发布者 | 订阅者 | 内容 |
|------|--------|--------|------|
| `execution:new` | main | worker | `{execution_id}` |
| `execution:{id}:update` | worker | main | trace span 增量 |
| `execution:{id}:complete` | worker | main | 最终状态 |

**备选方案（v1 简化）**：不用 Redis pub/sub，worker 轮询 DB 中 status=pending 的记录。main 的 WebSocket handler 轮询 trace.json 或 DB。代码更简单，延迟可接受（1-2 秒）。

## 四、Dockerfile

### 4.1 Backend + Worker (同一镜像，不同 CMD)

```dockerfile
# docker/backend/Dockerfile
FROM python:3.11-slim

WORKDIR /app

RUN apt-get update && apt-get install -y --no-install-recommends \
    curl \
    && rm -rf /var/lib/apt/lists/*

# Python 依赖
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 源码
COPY backend/ ./backend/

# 默认启动 main
CMD ["uvicorn", "backend.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

Worker 用同一个镜像，docker-compose 中覆盖 command：

```yaml
worker:
  build: ./docker/backend
  command: ["python", "-m", "backend.worker"]
  # ... 其他配置
```

### 4.2 Frontend

```dockerfile
# docker/frontend/Dockerfile
FROM node:20-alpine AS builder
WORKDIR /app
COPY frontend/package*.json ./
RUN npm ci
COPY frontend/ ./
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY docker/frontend/nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 4.3 Nginx 配置

```nginx
# docker/frontend/nginx.conf
upstream backend_main {
    server main:8000;
}

server {
    listen 80;
    server_name _;

    # 静态文件
    root /usr/share/nginx/html;
    index index.html;

    # API 反向代理
    location /api/ {
        proxy_pass http://backend_main;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 120s;  # 工作流可能耗时较长
    }

    # WebSocket 反向代理
    location /ws/ {
        proxy_pass http://backend_main;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 3600s;  # WebSocket 长连接
    }

    # SPA 回退
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

## 六、Worker 实现

```python
# backend/worker.py
import asyncio
import os
from tortoise import Tortoise
from config import TORTOISE_ORM
from loguru import logger
import aioredis

POLL_INTERVAL = int(os.getenv("WORKER_POLL_INTERVAL", "2"))
MAX_CONCURRENT = int(os.getenv("WORKER_MAX_CONCURRENT", "3"))

async def main():
    # 初始化 ORM
    await Tortoise.init(config=TORTOISE_ORM)

    # 连接 Redis
    redis = await aioredis.from_url(os.getenv("REDIS_URL"))

    # 信号量控制并发
    semaphore = asyncio.Semaphore(MAX_CONCURRENT)

    logger.info(f"Worker started (poll={POLL_INTERVAL}s, max_concurrent={MAX_CONCURRENT})")

    while True:
        # 领取待执行的工作流
        from models.workflow_trace import WorkflowTrace
        pending = await WorkflowTrace.filter(status="pending").first()

        if pending:
            async with semaphore:
                await execute_workflow(pending, redis)
        else:
            await asyncio.sleep(POLL_INTERVAL)


async def execute_workflow(trace, redis):
    """执行一个工作流实例"""
    from core.workflow_engine import workflow_engine
    from core.execution_tracer import ExecutionTracer

    # 标记为 running
    trace.status = "running"
    await trace.save()

    # 初始化追踪器
    tracer = ExecutionTracer(trace.execution_id, workspace_path_from(trace))
    tracer.start_execution(trace.workflow.name)

    try:
        # 加载工作流配置并执行
        result = await workflow_engine.execute(
            workflow_id=trace.workflow_id,
            execution_id=trace.execution_id,
            tracer=tracer
        )
        tracer.finish_execution("success" if result.success else "partial")

    except Exception as e:
        tracer.finish_execution("failed")
        trace.error_summary = str(e)

    # 更新 DB
    trace.status = tracer.trace["status"]
    trace.total_duration_ms = tracer.trace.get("total_duration_ms")
    trace.completed_at = datetime.utcnow()
    await trace.save()

    # 通知 main (WebSocket 推送)
    await redis.publish(f"execution:{trace.execution_id}:complete", json.dumps({
        "execution_id": trace.execution_id,
        "status": trace.status
    }))


if __name__ == "__main__":
    asyncio.run(main())
```

## 九、扩缩策略

### 水平扩展 Worker

```bash
# 临时扩容(高峰时)
docker compose up -d --scale worker=5

# 缩容
docker compose up -d --scale worker=1
```

Worker 是无状态的（状态全在 MySQL + workspace volume），可以安全扩缩。

### 不扩 main 的情况

main 是有状态的（WebSocket 连接）。需要扩时加 Redis 做 WebSocket 广播。v1 单实例 main 足够。

## 十、MCP Server 对接方式

平台不管理 MCP Server 进程。MCP Server 由各团队独立部署为 HTTP 服务，平台通过 `MCPClient` (HTTP) 对接调用。Worker 在执行工作流时发送 HTTP POST 到外部 MCP Server。

不需要在容器中预装 MCP Server 运行时（Node.js/Python 等），Worker 只需 `httpx` 即可。
