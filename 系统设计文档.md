## 一、项目概述

### 1.1 项目背景

研发团队日常工作中面临以下问题：
- 需要查询数据库、监控系统、K8s集群等多类线上数据
- 需要执行少量运维操作（重启、清理缓存等）
- 需要处理运营反馈、查阅流程文档
- 上述场景的Tools过多（10+），单一Agent难以准确选择工具

**扩展定位（v1）**：不仅是运维助手，而是覆盖**需求分析 → 方案设计 → 编码开发 → 代码审查 → 质量验证 → CI/CD部署**的完整 SDLC 端到端研发自动化平台。

### 1.2 项目目标

构建一个**面向研发团队内部使用的多Agent协同平台**，具备以下能力：
1. 支持配置多种LLM（OpenAI、Anthropic、Ollama等）
2. 支持创建三种类型 Agent：**local**（LangChain ReAct）、**http**（外部服务）、**claudecode**（Claude Code SDK）
3. 支持**平台级资源目录**：MCP Server 注册表、知识库管理、Skill 模板库，Agent 按需选用
4. 支持多Agent协同编排（监督者模式、流水线模式、图模式、并行组、人工中断）
5. 支持将编排好的工作流发布为应用，对外提供 Chat API
6. 支持端到端 SDLC 流程模板（需求→研发→测试→交付→部署）
7. 每个 Agent 拥有独立工作目录，支持代码仓库、CLAUDE.md 等隔离环境
8. 提供 Web 管理界面，供研发团队配置和使用

### 1.3 核心指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 单次对话响应时间（非流式） | < 5秒 | v1 使用 SQLite，目标 20-30 并发 |
| 流式响应首字延迟 | < 1秒 | |
| 支持并发会话数 | 20-30 | SQLite v1 验证阶段，后续可迁 PostgreSQL 扩展到 100+ |
| 系统可用性 | 99.5% | |
| SDLC 工作流完整执行 | < 30分钟 | 标准需求→部署全流程 |


## 二、技术架构

### 2.1 技术栈总览

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端框架** | Vue 3 + TypeScript | Composition API |
| **构建工具** | Vite | 快速开发 |
| **设计系统** | Linear Design System | 通过 getdesign.md 集成 DESIGN.md |
| **UI组件库** | Ant Design Vue | 企业级组件 |
| **工作流画布** | Vue Flow | 拖拽式编排 |
| **后端框架** | Python FastAPI | 高性能异步 |
| **Agent编排** | LangGraph | 多Agent协同核心 |
| **LLM接入** | LangChain + Anthropic SDK | 统一LLM接口 + Claude Code |
| **工具接入** | MCP Client (JSON-RPC) | 标准化工具协议 |
| **向量检索** | Chroma | 知识库 RAG |
| **数据库** | SQLite (v1) → PostgreSQL (扩展) | v1 验证用 SQLite，预留迁移路径 |
| **缓存/会话** | Redis | 会话状态、限流 |
| **加密** | Fernet (AES-128-CBC) | API Key 加密存储 |
| **部署** | Docker + Nginx | 容器化部署 |

### 3.3 平台资源目录（v1 新增）

#### 设计原则
- **资源与 Agent 解耦**：MCP Server、Skill、知识库作为平台级资源预先配置
- **Agent 按需选用**：创建 Agent 时从资源目录中选择，而非重复配置
- **支持工具级粒度**：选择 MCP Server 时可进一步筛选启用哪些 tools

```python
from typing import TypedDict, List, Any, Literal
from langgraph.graph import MessagesState

class AgentState(TypedDict):
    """LangGraph共享状态"""
    user_input: str                          # 用户原始输入
    messages: List[Any]                      # 对话历史
    current_agent: str                       # 当前执行的agent
    next_agent: str                          # 下一个agent（用于路由）
    intermediate_results: dict               # 中间结果存储 {agent_name: result}
    final_answer: str                        # 最终答案
    need_human: bool                         # 是否需要人工介入
    human_input: str                         # 人工输入
```

### 5.2 监督者模式实现

```python
from langgraph.graph import StateGraph, END
from langgraph.checkpoint import MemorySaver

def create_supervisor_workflow(supervisor_agent, worker_agents):
    """创建监督者模式工作流"""
    graph = StateGraph(AgentState)
    
    # 添加主管节点
    graph.add_node("supervisor", supervisor_agent)
    
    # 添加工作节点
    for name, agent in worker_agents.items():
        graph.add_node(name, agent)
        # 工作节点执行完后返回主管
        graph.add_edge(name, "supervisor")
    
    # 条件路由：主管决定下一个节点
    def route_supervisor(state: AgentState) -> Literal[*worker_agents.keys(), "end"]:
        next_agent = state.get("next_agent", "end")
        if next_agent == "end" or next_agent not in worker_agents:
            return "end"
        return next_agent
    
    graph.add_conditional_edges("supervisor", route_supervisor)
    graph.set_entry_point("supervisor")
    
    return graph.compile(checkpointer=MemorySaver())
```

### 5.3 Agent节点工厂

```python
from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic
from langgraph.prebuilt import create_react_agent

class AgentNodeFactory:
    """根据配置创建Agent节点"""
    
    def __init__(self, mcp_client: MCPClient):
        self.mcp_client = mcp_client
    
    def create(self, agent_config: dict):
        """创建单个Agent的执行函数"""
        # 1. 初始化LLM
        llm = self._init_llm(agent_config["llm_config"])
        
        # 2. 加载MCP工具
        tools = []
        for mcp_server in agent_config.get("mcp_servers", []):
            tools.extend(self.mcp_client.load_tools(mcp_server))
        
        # 3. 加载Skills（作为Prompt上下文）
        skills_context = self._load_skills(agent_config.get("skills", []))
        full_prompt = agent_config["system_prompt"] + "\n\n" + skills_context
        
        # 4. 创建ReAct Agent
        react_agent = create_react_agent(
            model=llm,
            tools=tools,
            prompt=full_prompt
        )
        
        # 5. 返回节点函数
        def agent_node(state: AgentState) -> dict:
            result = react_agent.invoke({
                "messages": state.get("messages", []),
                "user_input": state["user_input"]
            })
            return {
                "messages": result["messages"],
                "intermediate_results": {
                    **state.get("intermediate_results", {}),
                    agent_config["name"]: result["output"]
                }
            }
        
        return agent_node
```

### 5.4 MCP Client 完整实现

#### 5.4.1 MCP 协议概述

MCP (Model Context Protocol) 基于 JSON-RPC 2.0，使用 stdio 传输层。一个完整的 MCP 交互经历以下生命周期：

```
Client (Platform)                           MCP Server (Subprocess)
    │                                              │
    │── initialize ──────────────────────────────▶│  ① 握手：交换协议版本和能力
    │◀─ {protocolVersion, capabilities, ...} ─────│
    │                                              │
    │── initialized (notification) ──────────────▶│  ② 确认初始化完成
    │                                              │
    │── tools/list ──────────────────────────────▶│  ③ 发现工具列表
    │◀─ {tools: [{name, description, schema}]} ───│
    │                                              │
    │── tools/call {name, arguments} ────────────▶│  ④ 调用工具
    │◀─ {content: [{type: "text", text: "..."}]} ──│
    │                                              │
    │── tools/call ... ──────────────────────────▶│  (反复调用)
    │◀─ ... ──────────────────────────────────────│
    │                                              │
    │── (进程关闭或空闲超时) ─────────────────────▶│  ⑤ 清理
```

#### 5.4.2 进程生命周期管理

| 阶段 | 策略 | 说明 |
|------|------|------|
| 启动 | 惰性启动 (Lazy) | 首次调用工具时启动 MCP Server 进程 |
| 健康检查 | 心跳 Ping | 每 30s 发送 `ping` 请求，连续 3 次失败则重启 |
| 空闲回收 | TTL 300s | 300 秒无调用后关闭进程释放资源 |
| 崩溃恢复 | 自动重启 | 进程异常退出时自动重新初始化，最多重试 3 次 |
| 关闭 | 优雅终止 | SIGTERM → 等待 5s → SIGKILL |

#### 5.4.3 JSON-RPC 消息格式

**请求 (Request)**：
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "query_slow_log",
    "arguments": {
      "hours": 1,
      "limit": 10
    }
  }
}
```

**响应 (Response)**：
```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "找到3条慢查询:\n1. SELECT * FROM orders... 耗时2.3s\n..."
      }
    ]
  }
}
```

**错误响应**：

#### 5.4.4 完整实现代码

```python
import subprocess
import json
import asyncio
import time
import uuid
from typing import List, Dict, Any, Optional
from dataclasses import dataclass, field
from langchain_core.tools import tool
from loguru import logger


@dataclass
class MCPServerState:
    """MCP Server 运行时状态"""
    name: str
    process: subprocess.Popen
    stdin: Any  # 实际为 asyncio.StreamWriter
    stdout: Any  # 实际为 asyncio.StreamReader
    tools: List[Dict] = field(default_factory=list)
    request_id: int = 0
    last_used_at: float = field(default_factory=time.time)
    init_complete: bool = False


class MCPClient:
    """MCP Server 客户端 - 完整实现"""
    
    IDLE_TTL = 300        # 空闲超时（秒）
    HEARTBEAT_INTERVAL = 30  # 心跳间隔（秒）
    MAX_RESTART_ATTEMPTS = 3
    MAX_HEARTBEAT_FAILURES = 3
    
    def __init__(self):
        self._servers: Dict[str, MCPServerState] = {}
        self._cleanup_task: Optional[asyncio.Task] = None
    
    # ========== 1. 进程管理 ==========
    
    async def start_server(self, config: Dict[str, Any]) -> MCPServerState:
        """启动 MCP Server 并完成初始化握手"""
        name = config["name"]
        
        # 检查是否已在运行
        if name in self._servers:
            state = self._servers[name]
            if await self._health_check(state):
                state.last_used_at = time.time()
                return state
            else:
                await self._force_kill(name)
        
        # 启动进程
        process = await asyncio.create_subprocess_exec(
            config["command"], *config.get("args", []),
            stdin=asyncio.subprocess.PIPE,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
            env={**__import__("os").environ, **config.get("env", {})}
        )
        
        state = MCPServerState(
            name=name,
            process=process,
            stdin=process.stdin,
            stdout=process.stdout
        )
        
        # 执行初始化握手
        await self._initialize(state)
        
        # 发现工具
        state.tools = await self._list_tools(state)
        logger.info(f"MCP Server [{name}] 启动完成，提供 {len(state.tools)} 个工具")
        
        self._servers[name] = state
        
        # 启动后台清理任务（如果尚未启动）
        if self._cleanup_task is None:
            self._cleanup_task = asyncio.create_task(self._cleanup_loop())
        
        return state
    
    async def _initialize(self, state: MCPServerState) -> None:
        """MCP 初始化握手"""
        # 发送 initialize 请求
        response = await self._send_request(state, "initialize", {
            "protocolVersion": "2024-11-05",
            "capabilities": {
                "tools": {}  # 声明客户端支持工具调用
            },
            "clientInfo": {
                "name": "hh-agent-hub",
                "version": "1.0.0"
            }
        })
        
        protocol_version = response.get("protocolVersion")
        logger.debug(f"MCP Server [{state.name}] 协议版本: {protocol_version}")
        
        # 发送 initialized 通知
        await self._send_notification(state, "notifications/initialized", {})
        state.init_complete = True
    
    async def _health_check(self, state: MCPServerState) -> bool:
        """心跳检查"""
        if state.process.returncode is not None:
            return False
        try:
            await self._send_request(state, "ping", {}, timeout=5)
            return True
        except Exception:
            return False
    
    # ========== 2. 工具发现 ==========
    
    async def _list_tools(self, state: MCPServerState) -> List[Dict]:
        """发送 tools/list 获取工具清单"""
        response = await self._send_request(state, "tools/list", {})
        tools = response.get("tools", [])
        
        # 标准化工具定义
        normalized = []
        for t in tools:
            normalized.append({
                "name": t["name"],
                "description": t.get("description", ""),
                "inputSchema": t.get("inputSchema", {"type": "object", "properties": {}})
            })
        return normalized
    
    # ========== 3. 工具调用 ==========
    
    async def call_tool(self, server_name: str, tool_name: str, 
                        arguments: Dict[str, Any]) -> str:
        """调用 MCP 工具"""
        state = await self.start_server(
            {"name": server_name, "command": "", "args": [], "env": {}}
        )
        
        # 查找或重新获取 state（start_server 会返回正确的 state）
        state = self._servers[server_name]
        state.last_used_at = time.time()
        
        response = await self._send_request(state, "tools/call", {
            "name": tool_name,
            "arguments": arguments
        })
        
        # 解析响应内容
        content_parts = []
        for item in response.get("content", []):
            if item.get("type") == "text":
                content_parts.append(item["text"])
            elif item.get("type") == "resource":
                # 资源类型：可能有 uri + text 或 blob
                content_parts.append(item.get("text", "") or item.get("uri", ""))
        
        return "\n".join(content_parts)
    
    # ========== 4. JSON-RPC 通信 ==========
    
    async def _send_request(self, state: MCPServerState, method: str, 
                            params: Dict, timeout: int = 30) -> Dict:
        """发送 JSON-RPC 请求并等待响应"""
        state.request_id += 1
        request = {
            "jsonrpc": "2.0",
            "id": state.request_id,
            "method": method,
            "params": params
        }
        
        await self._write_message(state, request)
        
        # 读取响应（按 id 匹配）
        try:
            response = await asyncio.wait_for(
                self._read_response(state, state.request_id),
                timeout=timeout
            )
        except asyncio.TimeoutError:
            logger.error(f"MCP Server [{state.name}] 请求超时: {method}")
            raise TimeoutError(f"MCP call '{method}' timeout after {timeout}s")
        
        if "error" in response:
            err = response["error"]
            raise Exception(f"MCP Error [{err.get('code')}]: {err.get('message')}")
        
        return response.get("result", {})
    
    async def _send_notification(self, state: MCPServerState, method: str, 
                                  params: Dict) -> None:
        """发送 JSON-RPC 通知（无需响应，无 id）"""
        notification = {
            "jsonrpc": "2.0",
            "method": method,
            "params": params
        }
        await self._write_message(state, notification)
    
    async def _write_message(self, state: MCPServerState, message: Dict) -> None:
        """写入消息（每行一个 JSON，后跟换行符）"""
        line = json.dumps(message, ensure_ascii=False) + "\n"
        state.stdin.write(line.encode("utf-8"))
        await state.stdin.drain()
    
    async def _read_response(self, state: MCPServerState, 
                              expected_id: int) -> Dict:
        """读取匹配 id 的响应"""
        while True:
            line = await state.stdout.readline()
            if not line:
                raise ConnectionError(f"MCP Server [{state.name}] 连接关闭")
            
            message = json.loads(line.decode("utf-8"))
            
            # 跳过通知（无 id）
            if "id" not in message:
                continue
            
            if message["id"] == expected_id:
                return message
    
    # ========== 5. 资源清理 ==========
    
    async def _cleanup_loop(self) -> None:
        """后台清理循环：回收空闲进程 + 心跳检查"""
        while True:
            await asyncio.sleep(30)
            to_remove = []
            
            for name, state in list(self._servers.items()):
                # 检查进程是否存活
                if state.process.returncode is not None:
                    logger.warning(f"MCP Server [{name}] 已退出，清理")
                    to_remove.append(name)
                    continue
                
                # 空闲回收
                idle_time = time.time() - state.last_used_at
                if idle_time > self.IDLE_TTL:
                    logger.info(f"MCP Server [{name}] 空闲 {idle_time:.0f}s，回收")
                    await self._graceful_shutdown(name)
                    to_remove.append(name)
            
            for name in to_remove:
                self._servers.pop(name, None)
            
            if not self._servers:
                self._cleanup_task = None
                break
    
    async def _graceful_shutdown(self, name: str) -> None:
        """优雅关闭 MCP Server"""
        state = self._servers.get(name)
        if not state:
            return
        try:
            state.process.terminate()
            await asyncio.wait_for(state.process.wait(), timeout=5)
        except asyncio.TimeoutError:
            state.process.kill()
            await state.process.wait()
    
    async def _force_kill(self, name: str) -> None:
        """强制终止 MCP Server"""
        state = self._servers.pop(name, None)
        if state:
            try:
                state.process.kill()
                await state.process.wait()
            except Exception:
                pass
    
    async def shutdown_all(self) -> None:
        """关闭所有 MCP Server"""
        if self._cleanup_task:
            self._cleanup_task.cancel()
        for name in list(self._servers.keys()):
            await self._graceful_shutdown(name)
        self._servers.clear()
    
    # ========== 6. LangChain 工具适配 ==========
    
    async def load_tools(self, server_config: Dict) -> List:
        """将 MCP 工具转换为 LangChain Tool"""
        state = await self.start_server(server_config)
        
        langchain_tools = []
        for meta in state.tools:
            # 使用闭包捕获 server_name 和 tool_name
            def make_tool(server_name: str, tool_meta: Dict):
                @tool
                async def wrapped_tool(**kwargs) -> str:
                    return await self.call_tool(server_name, tool_meta["name"], kwargs)
                wrapped_tool.name = tool_meta["name"]
                wrapped_tool.description = tool_meta.get("description", "")
                # 设置 args_schema 来自 inputSchema（如果可用）
                return wrapped_tool
            
            langchain_tools.append(make_tool(state.name, meta))
        
        return langchain_tools
```

### 5.5 知识库检索与注入

#### 5.5.1 v1 策略：直接注入，不上向量库

知识库规模较小（几十个 Markdown 文档），v1 采用**按标题分块 + 关键词匹配 + 直接注入 Prompt**，不引入向量数据库。

```

    │
    ▼
目录扫描: 遍历 source_path 下所有匹配 file_patterns 的文件
    │
    ▼
Markdown 分块: 按 ## 标题拆分段落，保留标题层级路径
    │
    ▼
存储: content_blocks 表，每个块 = (kb_id, source_file, heading_path, body, token_count)
    │
    ▼
检索: 用户问题 → 关键词提取 → SQL LIKE 匹配标题+正文
    │
    ▼
排序: 匹配数 + 标题层级相似度
    │
    ▼
注入: Top-K chunk 拼入 Agent System Prompt 的 "## 参考资料" 区块
```

#### 5.5.3 Agent 运行时注入

Agent Factory 在创建节点时，根据 Agent 关联的知识库自动检索并注入上下文：

```python
class KnowledgeInjector:
    """知识库上下文注入器"""

    def __init__(self, retriever: KnowledgeRetriever):
        self.retriever = retriever

    async def inject(self, kb_ids: List[int], user_query: str,
                     base_prompt: str, max_tokens: int = 2000) -> str:
        """将知识库内容注入 Agent Prompt"""
        if not kb_ids:
            return base_prompt

        # 1. 检索相关 chunk
        results = await self.retriever.search(kb_ids, user_query, top_k=5)

        # 2. 控制 token 总量
        injected_content = []
        token_count = 0
        for r in results:
            estimated_tokens = len(r.content) // 2  # 粗略估算
            if token_count + estimated_tokens > max_tokens:
                break
            injected_content.append(f"### {r.heading_path}\n{r.content}")
            token_count += estimated_tokens

        # 3. 拼接
        if injected_content:
            base_prompt += "\n\n## 参考资料（来自知识库）\n" + "\n\n".join(injected_content)

        return base_prompt
```

### 5.7 工作流错误处理策略

#### 5.7.1 策略定义

每个工作流支持三种错误处理策略，在 `workflows` 表中通过 `error_strategy` 字段配置：

| 策略 | 值 | 行为 |
|------|------|------|
| **fail-fast** | `"fail_fast"` | 任一 Agent 失败，工作流立即终止，返回已收集的中间结果 |
| **skip-and-continue** | `"skip_continue"` | 失败 Agent 被跳过，标记为失败，后续 Agent 继续执行 |
| **retry** | `"retry"` | 失败 Agent 自动重试（最多 N 次），全部失败后再按 fail-fast 处理 |

#### 5.7.2 超时控制

```python
from dataclasses import dataclass
from typing import Optional
import asyncio

@dataclass
class WorkflowTimeoutConfig:
    """工作流超时配置"""
    agent_timeout_seconds: int = 60      # 单个 Agent 超时（秒）
    workflow_timeout_seconds: int = 300  # 整个工作流超时（秒）
    retry_count: int = 2                 # 重试次数
    retry_delay_seconds: int = 3         # 重试间隔（秒）

class WorkflowExecutor:
    """带错误处理的工作流执行器"""
    
    def __init__(self, config: WorkflowTimeoutConfig):
        self.config = config
        self.error_log: list[dict] = []  # 错误日志
    
    async def execute_with_timeout(self, agent_fn, state: dict, 
                                    agent_name: str) -> dict:
        """带超时的 Agent 执行"""
        try:
            result = await asyncio.wait_for(
                asyncio.to_thread(agent_fn, state),
                timeout=self.config.agent_timeout_seconds
            )
            return {"success": True, "result": result, "agent": agent_name}
        except asyncio.TimeoutError:
            self.error_log.append({
                "agent": agent_name,
                "error": "timeout",
                "message": f"超过 {self.config.agent_timeout_seconds}s 限制"
            })
            return {"success": False, "error": "timeout", "agent": agent_name}
        except Exception as e:
            self.error_log.append({
                "agent": agent_name,
                "error": str(e)
            })
            return {"success": False, "error": str(e), "agent": agent_name}
    
    async def execute_with_retry(self, agent_fn, state: dict, 
                                  agent_name: str) -> dict:
        """带重试的 Agent 执行"""
        last_error = None
        for attempt in range(self.config.retry_count + 1):
            result = await self.execute_with_timeout(agent_fn, state, agent_name)
            if result["success"]:
                result["retry_attempt"] = attempt
                return result
            last_error = result
            if attempt < self.config.retry_count:
                await asyncio.sleep(self.config.retry_delay_seconds)
        last_error["retry_exhausted"] = True
        return last_error
```

### 5.8 并行执行组与人工中断

#### 5.8.1 parallel_groups 设计

`parallel_groups` 字段支持在工作流中并行执行多个 Agent，适用于互不依赖的子任务：

**配置示例**：
```json
{
  "parallel_groups": [
    {
      "group_id": "pg1",
      "agents": [1, 2],
      "merge_strategy": "concat",
      "timeout_seconds": 120
    }
  ]
}
```

| 字段 | 说明 |
|------|------|
| group_id | 并行组唯一标识 |
| agents | 组内 Agent ID 列表 |
| merge_strategy | 结果合并策略：`concat`（拼接）/ `first`（取首个）/ `merge_by_key`（按键合并） |
| timeout_seconds | 并行组总超时（取组内最慢 Agent 完成时间） |

**执行流程**：
```
用户输入 → [Agent A, Agent B] 并行执行
              ├─ Agent A ──┐
              └─ Agent B ──┼──▶ 结果合并 → 下一节点
                           (两者都完成或超时)
```

**实现代码**：
```python
import asyncio
from typing import List, Dict, Any

async def execute_parallel_group(
    agents: List[tuple],  # [(name, node_fn), ...]
    state: Dict[str, Any],
    timeout: int = 120
) -> Dict[str, Any]:
    """并行执行一组 Agent"""
    tasks = []
    for agent_name, node_fn in agents:
        task = asyncio.create_task(
            asyncio.to_thread(node_fn, state),
            name=agent_name
        )
        tasks.append(task)
    
    # 等待所有完成或超时
    done, pending = await asyncio.wait(
        tasks, timeout=timeout
    )
    
    # 收集结果
    results = {}
    for task in done:
        agent_name = task.get_name()
        if task.exception():
            results[agent_name] = {"error": str(task.exception())}
        else:
            results[agent_name] = task.result()
    
    # 未完成的标记为超时
    for task in pending:
        task.cancel()
        results[task.get_name()] = {"error": "timeout"}
    
    return {"parallel_results": results}
```

### 5.9 Claude Code Agent 类型

#### 5.9.1 概述

Claude Code Agent 是继 local、http 之后的第三种 Agent 类型，专门集成 Claude Code 的能力。与 local 类型的区别在于：

| 维度 | local Agent | claudecode Agent |
|------|-------------|------------------|
| 核心引擎 | LangChain ReAct Agent | Claude Code SDK（文件读写+命令执行） |
| 工具集 | MCP + 自定义工具 | Claude Code 原生工具（Read/Write/Edit/Bash/Grep/Glob） + MCP |
| 适用场景 | 通用对话、工具调用 | 代码分析、文件操作、项目级任务 |
| 工作目录 | 无 | 支持指定项目工作目录 |
| 权限模型 | 无 | 支持 permission_mode 配置 |

#### 5.9.2 配置 Schema

```json
{
  "agent_type": "claudecode",
  "claudecode_config": {
    "model": "claude-sonnet-4-6",
    "max_turns": 25,
    "work_dir": "/home/user/projects/",
    "permission_mode": "acceptEdits",
    "claude_md_path": "/data/claude_configs/agent_1.md",
    "extra_args": ["--verbose"],
    "allowed_tools": ["Read", "Write", "Edit", "Bash", "Grep", "Glob"]
  }
}
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| model | string | 否 | `claude-sonnet-4-6` | Claude 模型版本 |
| max_turns | int | 否 | 25 | 最大对话轮次，防止死循环 |
| work_dir | string | 是 | - | Claude Code 的工作目录 |
| permission_mode | string | 否 | `default` | 权限模式：`default` / `acceptEdits` / `bypassPermissions` / `plan` |
| claude_md_path | string | 否 | - | CLAUDE.md 配置文件路径 |
| extra_args | string[] | 否 | [] | 传递额外的 CLI 参数 |
| allowed_tools | string[] | 否 | 全部 | 限制可用工具白名单 |
| env | object | 否 | {} | 环境变量（API Key 等） |

**权限模式说明**：

| 模式 | 行为 |
|------|------|
| `default` | 询问用户确认每个操作 |
| `acceptEdits` | 自动批准文件编辑，询问其他操作 |
| `bypassPermissions` | 完全自动执行（仅限受信任的工作目录） |
| `plan` | 仅规划不执行 |

#### 5.9.3 Claude Code SDK 集成

在环境中已集成 Claude Code SDK 的前提下，通过子进程调用 CLI 或直接使用 Anthropic SDK 实现 Agent：

```
                    ┌──────────────────────┐
                    │   AgentNodeFactory   │
                    └──────────┬───────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                ▼
        local Agent      http Agent      claudecode Agent
              │                │                │
              ▼                ▼                ▼
        LangChain         HTTP Client      ClaudeCodeRunner
        ReAct Agent                        │
                                    ┌──────┴──────┐
                                    ▼             ▼

```

**CLI 模式**（推荐用于 v1，无需额外 SDK）：
```python
import asyncio
import json
import os
from typing import Dict, Any

class ClaudeCodeRunner:
    """Claude Code CLI 执行器"""
    
    def __init__(self, config: Dict[str, Any]):
        self.work_dir = config.get("work_dir", os.getcwd())
        self.model = config.get("model", "claude-sonnet-4-6")
        self.max_turns = config.get("max_turns", 25)
        self.permission_mode = config.get("permission_mode", "acceptEdits")
        self.allowed_tools = config.get("allowed_tools", [])
        self.extra_args = config.get("extra_args", [])
        self.env = config.get("env", {})
    
    async def invoke(self, user_input: str, session_id: str, 
                     context: Dict[str, Any] = None) -> str:
        """通过 CLI 执行 Claude Code 任务"""
        
        # 构建 CLAUDE.md 临时内容（合并系统提示词 + 上下文）
        claude_md_content = self._build_claude_md(context)
        claude_md_path = self._write_temp_claude_md(claude_md_content, session_id)
        
        # 构建命令行参数
        cmd = [
            "claude",
            "--model", self.model,
            "--max-turns", str(self.max_turns),
            "--permission-mode", self.permission_mode,
            "--print",  # 非交互模式，直接输出结果
            "--output-format", "json",
            # 注入上下文
            "--custom-instructions", claude_md_path,
        ]
        
        # 添加工具白名单
        if self.allowed_tools:
            cmd.extend(["--allowed-tools", ",".join(self.allowed_tools)])
        
        cmd.extend(self.extra_args)
        
        # 执行
        process = await asyncio.create_subprocess_exec(
            *cmd,
            stdin=asyncio.subprocess.PIPE,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
            cwd=self.work_dir,
            env={**os.environ, **self.env}
        )
        
        stdout, stderr = await asyncio.wait_for(
            process.communicate(input=user_input.encode("utf-8")),
            timeout=self.max_turns * 15  # 每轮估算 15 秒
        )
        
        if process.returncode != 0:
            error_msg = stderr.decode("utf-8", errors="replace")
            raise RuntimeError(f"Claude Code 执行失败 (code={process.returncode}): {error_msg}")
        
        # 解析输出
        output = stdout.decode("utf-8", errors="replace")
        try:
            result = json.loads(output)
            return result.get("result", output)
        except json.JSONDecodeError:
            return output
    
    def _build_claude_md(self, context: Dict[str, Any] = None) -> str:
        """构建 CLAUDE.md 内容"""
        parts = []
        if context and context.get("system_prompt"):
            parts.append(context["system_prompt"])
        if context and context.get("intermediate_results"):
            parts.append(f"\n## 上游结果\n{json.dumps(context['intermediate_results'], ensure_ascii=False, indent=2)}")
        return "\n".join(parts) if parts else ""
    
    def _write_temp_claude_md(self, content: str, session_id: str) -> str:
        """写入临时 CLAUDE.md"""
        import tempfile
        tmp_dir = tempfile.mkdtemp(prefix=f"claude_md_{session_id}_")
        path = os.path.join(tmp_dir, "CLAUDE.md")
        if content:
            with open(path, "w", encoding="utf-8") as f:
                f.write(content)
        return path
```

**SDK 模式**（后续优化，利用 Anthropic SDK 直接调用）：
```python
# 当需要在工作流中更精细控制时，可通过 Anthropic SDK 构建
import anthropic

class ClaudeCodeSDKRunner:
    """基于 Anthropic SDK 的 Claude Code 执行器"""
    
    def __init__(self, config: Dict[str, Any]):
        self.client = anthropic.Anthropic(api_key=config.get("api_key"))
        self.model = config.get("model", "claude-sonnet-4-6")
        self.max_turns = config.get("max_turns", 25)
        # 注册 Claude Code 原生工具
        self.tools = [
            # Read, Write, Edit, Bash, Grep, Glob 等工具定义
        ]
    
    async def invoke(self, user_input: str, context: Dict = None) -> str:
        """在使用 Anthropic SDK 的多轮对话循环中执行任务"""
        messages = [{"role": "user", "content": user_input}]
        
        for turn in range(self.max_turns):
            response = self.client.messages.create(
                model=self.model,
                max_tokens=4096,
                system=context.get("system_prompt", "") if context else "",
                messages=messages,
                tools=self.tools
            )
            
            # 处理工具调用
            if response.stop_reason == "tool_use":
                tool_results = await self._execute_tools(response.content)
                messages.append({"role": "assistant", "content": response.content})
                messages.append({"role": "user", "content": tool_results})
            else:
                return response.content[0].text
        
        raise RuntimeError(f"超过最大轮次 {self.max_turns}")
```

### 5.10 工作流执行工作空间

#### 5.10.1 设计原则

每个工作流执行实例分配一个独立工作空间，所有参与 Agent 共享——不是 SDLC 专属能力，而是所有应用类型的通用基础设施。

- **通用性**：无论是研发 pipeline、线上问题排查还是代码审查，都使用同一套 workspace 机制
- **接力模式**：上游 Agent 产出 → 写入 workspace → 下游 Agent 读取，持续接力
- **两层上下文**：Prompt 传递摘要/决策 → 文件系统传递大体积产物（代码、报告、日志）

```
                ┌─────────────────────────────────────┐
                │         Workflow Workspace          │
                │  /data/workflow_workspaces/{exec}/  │
                │                                     │
                │  repos/          ← 共享代码仓库     │
                │  artifacts/      ← 各阶段产物目录    │
                │  context.json    ← 当前进度+摘要     │
                │  execution.log   ← 全流程日志        │
                └─────────────────────────────────────┘
                          ▲        │        ▲
                  ┌───────┘        │        └───────┐
                  ▼                ▼                ▼
             Agent A           Agent B           Agent C
          (读代码+写设计)  (读设计+改代码)   (读代码+跑测试)
```

#### 5.10.2 目录结构

```
/data/workflow_workspaces/{execution_id}/
│
├── repos/                           # 可选：共享代码仓库
│   └── {repo_name}/                 # Engine 负责 clone
│
├── artifacts/                       # Agent 产出物（按阶段隔离）
│   ├── 1_{first_agent}/
│   ├── 2_{second_agent}/
│   └── ...
│
├── context.json                     # 执行状态 + 上游摘要
└── execution.log                    # 结构化日志
```

**是否 clone 代码仓库取决于应用类型**：

| 应用类型 | repos/ | Agent 做什么 |
|---------|--------|-------------|
| SDLC 研发流程 | clone | designer 读代码做设计，developer 改代码，tester 跑测试 |
| 线上问题排查 | 不 clone | 分析 agent 读知识库，运维 agent 调 MCP 查 K8s/DB |
| 代码审查 | clone | reviewer agent 读代码，检查规范、安全、性能 |
| CI/CD 部署 | clone | builder agent 编译，deployer agent 发布 |

#### 5.10.3 context.json 协议

工作流执行期间唯一的状态文件，Agent 读取它了解上游发生了什么，Engine 通过它调度下一阶段。

```json
{
  "execution_id": "exec_abc123",
  "workflow_name": "线上问题排查-告警响应",
  "current_stage": 2,
  "status": "running",
  "upstream_summary": [
    {
      "stage": 1,
      "agent": "problem_analyzer",
      "status": "success",
      "summary": "告警来自 db-prod-03 实例连接数飙升，初步判断为慢查询堆积导致",
      "artifacts": ["analysis_report.md"]
    }
  ]
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| execution_id | string | 唯一执行 ID |
| workflow_name | string | 工作流名称 |
| current_stage | int | 当前执行到第几步（0-based） |
| status | string | running / completed / failed / paused |
| upstream_summary | array | 已完成阶段的摘要列表，供下游 Agent 快速了解上下文 |
| upstream_summary[N].agent | string | Agent 名称 |
| upstream_summary[N].summary | string | 1-2 句话说明该阶段做了什么和结论 |
| upstream_summary[N].artifacts | array | 产出物文件名列表（相对 artifacts/N_agent/） |

#### 5.10.4 Engine 职责

```python
class WorkflowWorkspaceManager:
    """工作流工作空间管理器 — 所有应用类型通用"""

    BASE_PATH = "/data/workflow_workspaces"

    async def setup(self, execution_id: str, repo_url: str = None) -> str:
        """
        Engine 启动工作流时调用，准备空间。
        是否 clone 仓库取决于应用类型，Engine 从 app 配置中获知。
        """
        ws = f"{self.BASE_PATH}/{execution_id}"
        os.makedirs(f"{ws}/artifacts", exist_ok=True)

        if repo_url:
            repo_dir = f"{ws}/repos/{self._repo_name(repo_url)}"
            await self._clone(repo_url, repo_dir)

        self._write_context(ws, {
            "execution_id": execution_id,
            "current_stage": 0,
            "status": "running",
            "upstream_summary": []
        })
        return ws

    async def stage_transition(self, ws: str, agent_name: str,
                                stage_num: int, result: dict):
        """一个阶段完成，更新 context.json，Engine 决定下一阶段"""
        context = self._read_context(ws)
        context["current_stage"] = stage_num + 1
        context["upstream_summary"].append({
            "stage": stage_num,
            "agent": agent_name,
            "status": result.get("status", "success"),
            "summary": result.get("summary", ""),
            "artifacts": result.get("artifacts", [])
        })
        self._write_context(ws, context)

    def build_agent_prompt(self, ws: str, agent_config: dict,
                           task_description: str) -> str:
        """为 Agent 构建包含 workspace 上下文的 System Prompt"""
        context = self._read_context(ws)

        summary_text = ""
        for s in context.get("upstream_summary", []):
            summary_text += f"- [{s['agent']}] {s['summary']}\n"
            if s.get("artifacts"):
                summary_text += f"  产出物: {', '.join(s['artifacts'])}\n"

        stage_dir = f"{context['current_stage'] + 1}_{agent_config['name']}"

        return f"""## 你的角色
{agent_config.get('system_prompt', '')}

## 当前任务
{task_description}

## 工作空间路径
{ws}

## 上游阶段摘要
{summary_text or '(无，这是第一个阶段)'}

## 工作空间说明
- 上游产物在 {ws}/artifacts/ 目录下，请先阅读了解上下文
- 如有代码仓库，在 {ws}/repos/ 目录下
- 你的产出物请写入 {ws}/artifacts/{stage_dir}/
- 完成后生成一个 summary（1-2句话），供下游 Agent 了解你这步做了什么
"""
```

#### 5.10.6 与 Agent 个人工作目录的关系

| | 工作流 workspace | Agent 个人 workspace |
|---|---|---|
| 路径 | `/data/workflow_workspaces/{exec}/` | `/data/agent_workspaces/agent_{id}/` |
| 生命周期 | 一次执行（执行结束可归档或清理） | 持久存在（Agent 被删除时清理） |
| 用途 | 流程中共享代码和产物 | Agent 专属配置（CLAUDE.md、Skills） |

Agent 执行时，工作目录指向个人 workspace，同时被注入工作流 workspace 路径作为共享上下文。

#### 5.10.7 清理策略

| 时机 | 动作 |
|------|------|
| 工作流成功完成 | 归档保留 7 天，随后清理 |
| 工作流失败 | 保留 workspace 供调试，30 天后清理 |
| 手动触发 | 管理员可通过 API 清理指定执行的工作空间 |
| 磁盘告警 | 后台任务监控磁盘使用率，超过 80% 时优先清理已完成的旧 workspace |


### 5.11 Agent 执行追踪与实时监控

#### 5.11.1 追踪层级

每个工作流执行自动生成一棵追踪树，三层结构：

```
Workflow Execution
  ├─ Agent Span (agent_name, stage, status, duration)
  │   ├─ LLM Call Span (model, prompt, response, tokens, duration)
  │   ├─ Tool Call Span (mcp_server, tool_name, args, result, duration)
  │   ├─ Knowledge Retrieval Span (query, kb_ids, result_count, duration)
  │   └─ Error Span (exception, stack_trace, retry_count)
  └─ Agent Span ...
```

#### 5.11.2 存储策略：文件为主 + DB 为索引

由于追踪树可能很大（多个 Agent、每个 Agent 多次 LLM/Tool 调用），全量存 SQLite 会导致表膨胀。采用**文件存追踪树 + DB 存元数据**的策略。

**文件**：追踪树完整 JSON 写入 workspace 目录，和产物放在一起：
```
/data/workflow_workspaces/{execution_id}/
├── trace.json           ← 完整追踪树
├── context.json
├── artifacts/
└── ...
```

**DB 表**：只存元数据，用于列表查询和快速过滤：

```sql
CREATE TABLE workflow_traces (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
```

#### 5.11.4 自动采集机制

通过 LangChain `BaseCallbackHandler` 自动采集，Agent 无需手动埋点：

```python
import time
import json
import os
from langchain.callbacks.base import BaseCallbackHandler
from loguru import logger

class ExecutionTracer:
    """执行追踪器 — 每个工作流执行一个实例"""

    def __init__(self, execution_id: str, workspace_path: str):
        self.execution_id = execution_id
        self.workspace_path = workspace_path
        self.trace = {
            "execution_id": execution_id,
            "workflow_name": "",
            "status": "running",
            "started_at": None,
            "spans": [],
            "errors": []
        }
        self._current_agent_span = None

    def start_execution(self, workflow_name: str):
        self.trace["started_at"] = self._now()
        self.trace["workflow_name"] = workflow_name
        self._flush()

    def start_agent(self, agent_name: str, agent_type: str, stage: int):
        span = {
            "span_id": f"span_agent_{stage}",
            "type": "agent",
            "agent_name": agent_name,
            "agent_type": agent_type,
            "stage": stage,
            "status": "running",
            "started_at": self._now(),
            "children": []
        }
        self.trace["spans"].append(span)
        self._current_agent_span = span
        self._flush()

    def add_llm_span(self, model: str, provider: str, prompt: str,
                      response: str, input_tokens: int, output_tokens: int,
                      duration_ms: int, status: str):
        span = {
            "span_id": self._next_span_id(),
            "type": "llm_call",
            "model": model, "provider": provider,
            "prompt": prompt, "response": response,
            "input_tokens": input_tokens, "output_tokens": output_tokens,
            "duration_ms": duration_ms, "status": status
        }
        self._current_agent_span["children"].append(span)
        self._flush()

    def add_tool_span(self, tool_name: str, mcp_server: str,
                       arguments: dict, result: str,
                       duration_ms: int, status: str):
        span = {
            "span_id": self._next_span_id(),
            "type": "tool_call",
            "tool_name": tool_name, "mcp_server": mcp_server,
            "arguments": arguments, "result": result,
            "duration_ms": duration_ms, "status": status
        }
        self._current_agent_span["children"].append(span)
        self._flush()

    def add_kb_span(self, query: str, kb_ids: list,
                     result_count: int, duration_ms: int):
        span = {
            "span_id": self._next_span_id(),
            "type": "knowledge_retrieval",
            "query": query, "kb_ids": kb_ids,
            "result_count": result_count,
            "duration_ms": duration_ms, "status": "success"
        }
        self._current_agent_span["children"].append(span)
        self._flush()

    def finish_agent(self, status: str, summary: str):
        agent = self._current_agent_span
        agent["status"] = status
        agent["duration_ms"] = self._elapsed_ms(agent["started_at"])
        agent["summary"] = summary
        agent["token_usage"] = self._sum_tokens(agent["children"])
        self._flush()

    def finish_execution(self, status: str):
        self.trace["status"] = status
        self.trace["completed_at"] = self._now()
        self.trace["total_duration_ms"] = self._elapsed_ms(
            self.trace["started_at"])
        self._flush()

    def _flush(self):
        """写入 trace.json 并推送 WebSocket 事件"""
        path = os.path.join(self.workspace_path, "trace.json")
        with open(path, "w", encoding="utf-8") as f:
            json.dump(self.trace, f, ensure_ascii=False, indent=2)

        # 推送实时更新
        from core.ws_manager import ws_manager
        asyncio.create_task(
            ws_manager.push(self.execution_id, {
                "type": "trace_update",
                "trace": self.trace
            })
        )


class TraceCallback(BaseCallbackHandler):
    """LangChain Callback → 自动生成追踪 span"""

    def __init__(self, tracer: ExecutionTracer, model: str, provider: str):
        self.tracer = tracer
        self.model = model
        self.provider = provider
        self._start_time = None
        self._prompt = None

    def on_llm_start(self, serialized, prompts, **kwargs):
        self._start_time = time.time()
        self._prompt = prompts[0] if prompts else ""

    def on_llm_end(self, response, **kwargs):
        duration_ms = int((time.time() - self._start_time) * 1000)
        usage = response.llm_output.get("token_usage", {})
        self.tracer.add_llm_span(
            model=self.model,
            provider=self.provider,
            prompt=self._prompt,
            response=response.generations[0][0].text,
            input_tokens=usage.get("prompt_tokens", 0),
            output_tokens=usage.get("completion_tokens", 0),
            duration_ms=duration_ms,
            status="success"
        )

    def on_tool_start(self, serialized, input_str, **kwargs):
        self._tool_start = time.time()

    def on_tool_end(self, output, **kwargs):
        duration_ms = int((time.time() - self._tool_start) * 1000)
        # 工具信息从 serialized 中提取
        self.tracer.add_tool_span(
            tool_name=kwargs.get("name", "unknown"),
            mcp_server=kwargs.get("mcp_server", ""),
            arguments=kwargs.get("arguments", {}),
            result=str(output),
            duration_ms=duration_ms,
            status="success"
        )
```

#### 5.11.5 WebSocket 实时推送

```python
from fastapi import WebSocket
from typing import Dict, Set
import json

class WorkflowWebSocketManager:
    """工作流执行 WebSocket 管理器"""

    def __init__(self):
        # execution_id → {websocket, ...}
        self._subscribers: Dict[str, Set[WebSocket]] = {}

    async def subscribe(self, execution_id: str, ws: WebSocket):
        await ws.accept()
        self._subscribers.setdefault(execution_id, set()).add(ws)

    def unsubscribe(self, execution_id: str, ws: WebSocket):
        subs = self._subscribers.get(execution_id)
        if subs:
            subs.discard(ws)

    async def push(self, execution_id: str, event: dict):
        """推送事件到所有订阅该 execution 的客户端"""
        subs = self._subscribers.get(execution_id, set())
        dead = set()
        for ws in subs:
            try:
                await ws.send_json(event)
            except Exception:
                dead.add(ws)
        subs -= dead

ws_manager = WorkflowWebSocketManager()


# FastAPI WebSocket 端点
@router.websocket("/ws/execution/{execution_id}/live")
async def watch_execution(websocket: WebSocket, execution_id: str):
    await ws_manager.subscribe(execution_id, websocket)
    try:
        while True:
            # 保持连接，等待客户端消息（心跳或取消）
            await websocket.receive_text()
    except Exception:
        pass
    finally:
        ws_manager.unsubscribe(execution_id, websocket)
```

WebSocket 事件类型：

| 事件 | 触发时机 | payload |
|------|---------|---------|
| `execution_started` | 工作流启动 | `{execution_id, workflow_name, agent_count}` |
| `agent_started` | Agent 开始执行 | `{agent_name, stage, started_at}` |
| `span_completed` | 任一字 span 完成 | `{span_id, type, duration_ms, status}` |
| `agent_completed` | Agent 完成 | `{agent_name, status, summary, duration_ms}` |
| `execution_completed` | 工作流完成 | `{execution_id, status, total_duration_ms}` |
| `execution_error` | 工作流异常 | `{execution_id, error_agent, error_message}` |

### 7.1 SQLite部署

SQLite 作为嵌入式数据库，无需独立服务，数据存储在单个文件中，部署简单。

#### 数据库文件位置

```
/data/agent_platform.db
```

#### 初始化脚本 (init_db.py)

```python
import sqlite3
import os

def init_database(db_path: str = "/data/agent_platform.db"):
    """初始化SQLite数据库"""
    
    # 确保目录存在
    os.makedirs(os.path.dirname(db_path), exist_ok=True)
    
    conn = sqlite3.connect(db_path)
    cursor = conn.cursor()
    
    # 创建系统配置表
    cursor.execute('''
    CREATE TABLE IF NOT EXISTS sys_configs (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        config_key VARCHAR(100) NOT NULL UNIQUE,
        config_value TEXT NOT NULL,
        config_type VARCHAR(20) DEFAULT 'string',
        description VARCHAR(200),
        updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    )
    ''')
    
    # 创建用户表
    cursor.execute('''
    CREATE TABLE IF NOT EXISTS users (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        username VARCHAR(50) NOT NULL UNIQUE,
        password_hash VARCHAR(255) NOT NULL,
        api_key VARCHAR(64) NOT NULL UNIQUE,
        role VARCHAR(20) DEFAULT 'user',
        email VARCHAR(100),
        avatar VARCHAR(200),
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        last_login TIMESTAMP
    )
    ''')
    
    # 创建Agent表
    cursor.execute('''
    CREATE TABLE IF NOT EXISTS agents (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name VARCHAR(50) NOT NULL UNIQUE,
        display_name VARCHAR(100),
        description TEXT,
        role VARCHAR(20) NOT NULL,
        agent_type VARCHAR(20) DEFAULT 'local',
        llm_config TEXT,
        http_config TEXT,
        system_prompt TEXT,
        mcp_servers TEXT DEFAULT '[]',
        skills TEXT DEFAULT '[]',
        custom_tools TEXT DEFAULT '[]',
        status VARCHAR(20) DEFAULT 'active',
        created_by INTEGER REFERENCES users(id),
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        updated_at TIMESTAMP
    )
    ''')
    
    # 创建工作流表
    cursor.execute('''
    CREATE TABLE IF NOT EXISTS workflows (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name VARCHAR(100) NOT NULL,
        description TEXT,
        flow_type VARCHAR(20) NOT NULL,
        supervisor_agent_id INTEGER REFERENCES agents(id),
        worker_agent_ids TEXT DEFAULT '[]',
        edges TEXT DEFAULT '[]',
        parallel_groups TEXT DEFAULT '[]',
        human_interrupts TEXT DEFAULT '[]',
        status VARCHAR(20) DEFAULT 'draft',
        version INTEGER DEFAULT 1,
        created_by INTEGER REFERENCES users(id),
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        updated_at TIMESTAMP
    )
    ''')
    
    # 创建应用表
    cursor.execute('''
    CREATE TABLE IF NOT EXISTS apps (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        name VARCHAR(100) NOT NULL,
        description TEXT,
        workflow_id INTEGER NOT NULL REFERENCES workflows(id),
        workflow_version INTEGER DEFAULT 1,
        api_key VARCHAR(64) NOT NULL UNIQUE,
        rate_limit INTEGER DEFAULT 60,
        enabled BOOLEAN DEFAULT 1,
        created_by INTEGER REFERENCES users(id),
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        updated_at TIMESTAMP
    )
    ''')
    
    # 创建会话表
    cursor.execute('''
    CREATE TABLE IF NOT EXISTS sessions (
        id VARCHAR(36) PRIMARY KEY,
        app_id INTEGER REFERENCES apps(id),
        user_id VARCHAR(100),
        messages TEXT DEFAULT '[]',
        thread_state TEXT,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        updated_at TIMESTAMP,
        expired_at TIMESTAMP
    )
    ''')
    
    # 创建审计日志表
    cursor.execute('''
    CREATE TABLE IF NOT EXISTS audit_logs (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        user_id INTEGER REFERENCES users(id),
        action VARCHAR(50) NOT NULL,
        target_type VARCHAR(50),
        target_id INTEGER,
        request_id VARCHAR(36),
        ip_address VARCHAR(45),
        details TEXT,
        created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    )
    ''')
    
    # 创建索引
    cursor.execute('CREATE INDEX IF NOT EXISTS idx_users_username ON users(username)')
    cursor.execute('CREATE INDEX IF NOT EXISTS idx_users_api_key ON users(api_key)')
    cursor.execute('CREATE INDEX IF NOT EXISTS idx_agents_name ON agents(name)')
    cursor.execute('CREATE INDEX IF NOT EXISTS idx_agents_status ON agents(status)')
    cursor.execute('CREATE INDEX IF NOT EXISTS idx_workflows_status ON workflows(status)')
    cursor.execute('CREATE INDEX IF NOT EXISTS idx_apps_api_key ON apps(api_key)')
    cursor.execute('CREATE INDEX IF NOT EXISTS idx_sessions_app_id ON sessions(app_id)')
    cursor.execute('CREATE INDEX IF NOT EXISTS idx_sessions_expired_at ON sessions(expired_at)')
    cursor.execute('CREATE INDEX IF NOT EXISTS idx_audit_logs_user_id ON audit_logs(user_id)')
    cursor.execute('CREATE INDEX IF NOT EXISTS idx_audit_logs_created_at ON audit_logs(created_at)')
    
    # 插入默认配置
    default_configs = [
        ('llm.default.provider', 'openai', 'string', '默认LLM提供商'),
        ('llm.default.model', 'gpt-4o-mini', 'string', '默认模型'),
        ('llm.default.temperature', '0.3', 'number', '默认温度参数'),
        ('llm.default.api_key', '', 'secret', '默认API密钥'),
        ('system.max_tokens', '4096', 'number', '最大token限制'),
        ('system.session_ttl', '3600', 'number', '会话过期时间(秒)'),
        ('system.rate_limit.default', '60', 'number', '默认限流(次/分钟)')
    ]
    
    for config in default_configs:
        cursor.execute('''
        INSERT OR IGNORE INTO sys_configs (config_key, config_value, config_type, description)
        VALUES (?, ?, ?, ?)
        ''', config)
    
    conn.commit()
    conn.close()
    print(f"数据库初始化完成: {db_path}")

if __name__ == "__main__":
    init_database()
```

### 7.3 备份策略

SQLite 数据库备份简单，直接复制数据库文件即可：

```bash
# 备份脚本 backup.sh
#!/bin/bash
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backup"
DB_FILE="/data/agent_platform.db"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 使用 SQLite 在线备份 API（推荐）
sqlite3 $DB_FILE ".backup '${BACKUP_DIR}/agent_platform_${DATE}.db'"

# 或者简单复制
```

## 八、安全设计

### 8.1 API Key 加密存储

敏感字段（LLM API Key、应用 API Key 等）使用 Fernet 对称加密方案：

```python
from cryptography.fernet import Fernet
import os

class CryptoManager:
    """加密管理器"""
    
    def __init__(self):
        # 加密密钥从环境变量读取，不存在时自动生成
        self._key = os.getenv("FERNET_KEY")
        if not self._key:
            self._key = Fernet.generate_key().decode()
            os.environ["FERNET_KEY"] = self._key
        self._fernet = Fernet(self._key.encode() if isinstance(self._key, str) else self._key)
    
    def encrypt(self, plaintext: str) -> str:
        """加密"""
        return self._fernet.encrypt(plaintext.encode()).decode()
    
    def decrypt(self, ciphertext: str) -> str:
        """解密"""
        return self._fernet.decrypt(ciphertext.encode()).decode()

# 使用示例
crypto = CryptoManager()
encrypted_key = crypto.encrypt("sk-xxxx")  # 存储到数据库
decrypted_key = crypto.decrypt(encrypted_key)  # 运行时使用
```

**密钥管理规则**：
| 项目 | 策略 |
|------|------|
| 加密算法 | Fernet (AES-128-CBC + HMAC) |
| 密钥存储 | 环境变量 `FERNET_KEY`，不写入代码或数据库 |
| 密钥轮换 | 提供 `POST /system/crypto/rotate` API，用新密钥重新加密所有敏感字段 |
| 日志脱敏 | `loguru` 配置 `patch` 函数，自动替换 `sk-*`、`Bearer *` 等模式为 `***` |

### 8.2 数据库文件安全

```
正确路径: /data/agent_platform.db      (独立数据目录)
错误路径: ./backend/data/agent_platform.db  (在 web 可访问路径下)
```

- 数据库文件禁止放在源码目录或 web 静态资源目录下
- Docker 部署时通过 volume 挂载到宿主机，确保容器重建后数据不丢失

### 8.3 Prompt 注入风险

| 风险场景 | 缓解措施 |
|------|------|
| 用户在 Web UI 编辑 System Prompt 时注入恶意指令 | System Prompt 编辑需管理员权限，变更记录审计日志 |
| 外部用户通过 Chat API 注入指令覆盖 System Prompt | Chat API 入口对用户输入做角色隔离（user message 与 system message 分开注入） |
| MCP 工具返回值包含指令注入 | MCP 工具输出标记为 `tool_output` 角色，LLM 应区分对待 |
| Skill 文件被篡改 | Skill 文件路径只读，仅管理员可更新 |

### 8.4 API 安全

| 项目 | 措施 |
|------|------|
| 管理 API 认证 | JWT Token（7天过期）+ 角色鉴权 |
| Chat API 认证 | `X-API-Key` Header 或 `Authorization: Bearer` |
| 限流 | Redis 令牌桶，默认 60次/分钟，可按应用配置 |
| CORS | 生产环境白名单，开发环境允许 `localhost` |
| 请求大小限制 | 最大 10MB（FastAPI middleware） |

## 十、端到端研发流程 (SDLC Pipeline)

### 10.1 设计理念

平台不仅是"运维助手"，而是面向**完整研发生命周期**的自动化编排引擎。通过配置不同角色的 Agent 和编排工作流，实现从需求到交付的全流程自动化。

```
需求提出 ──▶ 需求分析Agent ──▶ 方案设计Agent ──▶ 编码开发Agent
                  │                    │               │
                  ▼                    ▼               ▼
             生成需求文档        生成技术方案      生成代码/PR
                                                       │
                                                       ▼
部署上线 ◀── CI/CD部署Agent ◀── 质量验证Agent ◀── 代码审查Agent
   │               │                  │               │
   ▼               ▼                  ▼               ▼
生产环境      自动构建发布        自动化测试      代码Review
```

### 10.2 研发流程中的 Agent 角色

| 阶段 | Agent 角色 | Agent 类型 | 环境/工具 |
|------|-----------|-----------|----------|
| **需求分析** | `requirement_analyst` | local | TAPD/Jira MCP, 文档 Skills |
| **方案设计** | `solution_designer` | claudecode | 架构文档 Skills, 设计模式库 |
| **编码开发** | `developer` | claudecode | 代码仓库工作目录, IDE MCP |
| **代码审查** | `code_reviewer` | claudecode | 代码仓库, Linter MCP |
| **质量验证** | `qa_tester` | claudecode | 测试框架, 测试用例库 |
| **CI/CD 部署** | `cicd_deployer` | local / http | K8s MCP, Docker MCP, Jenkins API |
| **运维监控** | `ops_monitor` | local | Prometheus MCP, 日志 MCP, 告警 MCP |

### 10.3 Agent 工作目录与工作流工作空间

SDLC 流程同时使用两层工作空间，详见 [5.10 工作流执行工作空间](#510-工作流执行工作空间)：

| 层 | 路径 | 用途 |
|----|------|------|
| Agent 个人工作目录 | `/data/agent_workspaces/agent_{id}/` | CLAUDE.md、Skills、个人配置 |
| 工作流执行工作空间 | `/data/workflow_workspaces/{exec_id}/` | repos/（clone 的代码）+ artifacts/（各阶段产物） |

工作流 Engine 启动时自动 clone 代码仓库到 workspace 的 `repos/` 目录，各阶段 Agent 在同一份代码上接力——designer 读代码写设计，developer 在同一份代码上开发，tester 在同一份代码上跑测试。

```python
import asyncio
import json
from unittest.mock import AsyncMock

class MockMCPServer:
    """模拟 MCP Server 用于测试"""
    
    def __init__(self, tools: list[dict]):
        self.tools = tools
    
    async def serve(self, host="localhost", port=0):
        """启动模拟 MCP Server"""
        server = await asyncio.start_server(
            self._handle_client, host, port
        )
        return server
    
    async def _handle_client(self, reader, writer):
        """处理 JSON-RPC 请求"""
        while True:
            line = await reader.readline()
            if not line:
                break
            request = json.loads(line.decode())
            
            method = request.get("method", "")
            
            if method == "initialize":
                response = self._init_response(request)
            elif method == "tools/list":
                response = self._list_tools_response(request)
            elif method == "tools/call":
                response = self._call_tool_response(request)
            else:
                response = self._error_response(request, -32601, "Method not found")
            
            writer.write((json.dumps(response) + "\n").encode())
            await writer.drain()
    
    def _init_response(self, request):
        return {
            "jsonrpc": "2.0",
            "id": request["id"],
            "result": {
                "protocolVersion": "2024-11-05",
                "capabilities": {"tools": {}}
            }
        }
```
